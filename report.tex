\section{Workflow of main.sh (The authentication system}
main.sh is the login page. It starts with the Mini-game-hub logo followed by getting credentials from player 1 first followed by player 2. The main authentication logic is as follows:
\\
\begin{itemize}[leftmargin=1.5em, topsep=0pt, itemsep=0pt]
    \\
    \item Firstly player 1's inputted username is checked for validity, i.e. doe sit have valid characters or not. The user is even informed if there is an issue with the username they inputted. \\
    \item Then player 1 is constantly asked for username until they either have the username already registered, i.e. the username they inputted is present as the one of the first entries in the user.tsv or they are willing to add the input as a new username. We extract the first column by the "cut" command after matching the username by the "grep" command and extracting the row. \\
    \item After that, the password inputted  is converted by hashing and compared to the password stores int he user.tsv corresponding to the username. The lop continues if the passwords don't match and terminates otherwise giving the appropriate feedback in either case.
    \\
    \item The same procedure is followed for player 2 and the game continues after both the user have been authorized. 
    \\
    \item Finally, the ending screen and the loading prompt is shown to both the users indicating the start of the game. 
    \\
\end{itemize}
